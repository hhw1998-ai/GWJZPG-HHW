# 高唐县财信投资发展集团岗位价值评估系统

## 一、系统概述

### 1.1 项目背景

岗位价值评估是现代人力资源管理的核心工具之一，通过科学的评估方法对岗位进行价值量化，为薪酬体系设计、岗位晋升通道构建提供客观依据。本系统基于国际通用的"六因素十四维度"评估模型，实现了评估流程的数字化、标准化和自动化。

### 1.2 系统定位

本系统是一套**企业级岗位价值评估解决方案**，适用于：
- 集团型企业的多公司、多岗位评估场景
- 需要多人独立评分、汇总统计的评估项目
- 追求评估过程可追溯、结果可导出的规范管理

### 1.3 核心价值

| 传统方式 | 本系统 |
|---------|-------|
| 纸质评分表，手动计算 | 在线评分，自动汇总 |
| 单人评估，主观性强 | 多人独立评分，平均计算 |
| 数据分散，难以追溯 | 数据集中，全程留痕 |
| 修改后难以发现 | 提交锁定，解锁需管理员授权 |
| 导出需手动整理 | 一键导出Excel多分页报表 |

### 3.3 评分标准特点

1. **千分制评分**：总分1000分，便于直观理解岗位价值差异
2. **差异等级数**：部分维度为4级（问题解决要求、外部沟通），其余为5级
3. **非线性分值**：等级分值非均匀分布，高等级分值增幅更大，体现岗位价值差异
4. **独立评分**：每个维度独立评分，互不影响

### 5.1 评分计算算法

```typescript
// 评分计算逻辑
function calculateTotalScore(scores: Record<string, number>): number {
  let totalScore = 0;
  
  // 遍历所有维度
  for (const dimension of allDimensions) {
    const level = scores[dimension.fieldName] || 0;
    if (level > 0) {
      // 根据等级获取对应分值
      totalScore += getScoreByLevel(dimension, level);
    }
  }
  
  return totalScore; // 返回千分制总分
}

// 根据维度和等级获取分值
function getScoreByLevel(dimension: Dimension, level: number): number {
  const levelData = dimension.levels.find(l => l.level === level);
  return levelData?.score || 0;
}
```

### 5.2 岗位排名算法

```typescript
// 岗位排名逻辑
function calculateRankings(evaluations: Evaluation[], positions: Position[]): Ranking[] {
  // 1. 按岗位分组评分
  const evaluationsByPosition = groupBy(evaluations, 'position_id');
  
  // 2. 计算各岗位平均分
  const rankings = positions.map(position => {
    const positionEvaluations = evaluationsByPosition[position.id] || [];
    
    if (positionEvaluations.length === 0) return null;
    
    // 计算平均分
    const totalScore = positionEvaluations.reduce((sum, e) => sum + e.total_score, 0);
    const averageScore = Math.round(totalScore / positionEvaluations.length);
    
    return {
      position,
      averageScore,
      evaluationCount: positionEvaluations.length,
      evaluators: getEvaluatorNames(positionEvaluations)
    };
  }).filter(Boolean);
  
  // 3. 排序：按平均分从高到低
  rankings.sort((a, b) => {
    if (b.averageScore !== a.averageScore) {
      return b.averageScore - a.averageScore; // 分数高的排前面
    }
    // 分数相同时，按公司顺序
    return a.companyOrder - b.companyOrder;
  });
  
  return rankings;
}
```

## 八、性能优化

### 8.1 Supabase 1000行限制优化

**问题**：Supabase 单次查询最多返回 1000 行，评分数据超过限制时会被截断。

**解决方案**：使用分页查询（range）循环获取所有数据。

```typescript
// 优化前（会被截断）
const { data } = await client
  .from('evaluations')
  .select('*')
  .limit(9999);  // 无效，仍只返回1000行

// 优化后（分页获取全部）
let allEvaluations: any[] = [];
let page = 0;
const pageSize = 1000;

while (true) {
  const { data: pageData } = await client
    .from('evaluations')
    .select('*')
    .range(page * pageSize, (page + 1) * pageSize - 1);
  
  if (!pageData || pageData.length === 0) break;
  
  allEvaluations = allEvaluations.concat(pageData);
  
  if (pageData.length < pageSize) break;  // 最后一页
  page++;
}
```

### 8.2 批量写入优化

**问题**：25人并发写入时成功率仅 72%，部分请求超时失败。

**解决方案**：

| 优化项 | 优化前 | 优化后 |
|--------|--------|--------|
| 评估人创建 | 先查后插（2次调用） | upsert（1次调用） |
| 获取现有评分 | 循环单查（N次调用） | 批量 in 查询（1次调用） |
| 写入操作 | 串行执行 | Promise.all 并行执行 |
| 数据库连接 | 每次新建连接 | 复用单例连接 |
| 数据库索引 | 无 | 添加关键字段索引 |

**效果**：成功率提升至 100%，平均耗时降低 55%。

### 9.2 提交锁定机制

```typescript
// 提交评分时设置锁定标志
const { data: existingSubmitted } = await client
  .from('evaluations')
  .select('id')
  .eq('evaluator_id', evaluatorId)
  .eq('submitted', true)
  .limit(1);

if (existingSubmitted && existingSubmitted.length > 0) {
  return NextResponse.json({ 
    error: '您已提交过评分，无法重复提交。如需修改请联系管理员。' 
  }, { status: 403 });
}

// 标记为已提交
await client
  .from('evaluations')
  .update({ submitted: true })
  .eq('evaluator_id', evaluatorId);
```

## 十二、常见问题

### Q1: 评分数据丢失怎么办？

**A**: 系统自动保存机制会在评分变化后 3 秒自动保存。如遇网络问题，刷新页面后会自动加载已保存的数据。

### Q2: 提交后如何修改评分？

**A**: 需联系管理员，通过后台"解锁管理"功能解锁后才能修改。

### Q3: Excel导入失败怎么办？

**A**: 检查Excel格式是否符合要求，系统会生成详细的错误报告，按报告修正后重新导入。

### Q4: 如何添加新公司？

**A**: 两种方式：
1. 通过Excel导入，系统自动识读新公司
2. 后台手动添加公司

### Q5: 如何导出评估数据？

**A**: 后台管理页面点击"导出Excel报表"，生成包含排名总表、详细评分等多分页的Excel文件。
